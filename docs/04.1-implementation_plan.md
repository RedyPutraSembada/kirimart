# Rencana Implementasi Skema Database (E-Commerce MVP)

Tujuan: Melaksanakan **Step 4** dari MVP Plan, yaitu menerjemahkan desain ERD ke dalam skema Drizzle ORM (PostgreSQL) yang siap digunakan.

*Catatan: Sesuai permintaan Anda, seluruh Primary Key akan menggunakan tipe `serial` (integer auto-increment) dan setiap tabel akan dipisah ke dalam satu file tersendiri.*

## Proposed Changes

Kita akan membuat 15 file skema baru di dalam direktori `src/config/db/schema/`, di mana **1 file = 1 tabel**.

### Schema Files (1 Tabel = 1 File)

1. `address-schema.js` (Tabel `addresses`)
2. `store-schema.js` (Tabel `stores`)
3. `category-schema.js` (Tabel `categories`)
4. `product-schema.js` (Tabel `products`)
5. `product-image-schema.js` (Tabel `product_images`)
6. `cart-schema.js` (Tabel `carts`)
7. `cart-item-schema.js` (Tabel `cart_items`)
8. `voucher-schema.js` (Tabel `vouchers`)
9. `payment-schema.js` (Tabel `payments`)
10. `order-schema.js` (Tabel `orders`)
11. `order-item-schema.js` (Tabel `order_items`)
12. `shipment-schema.js` (Tabel `shipments`)
13. `review-schema.js` (Tabel `reviews`)
14. `conversation-schema.js` (Tabel `conversations`)
15. `message-schema.js` (Tabel `messages`)

### Core Configuration

#### [MODIFY] `relations.js`
- Menghubungkan seluruh relasi Drizzle untuk 15 tabel tersebut dalam satu file terpusat. Memusatkan relasi di satu file adalah *best practice* di Drizzle ORM untuk mencegah *Circular Dependency* (kegagalan impor bolak-balik antar file).

#### [MODIFY] `index.js`
- Mengekspor secara rata (*flat export*) semua 15 skema baru ini agar Drizzle Kit mendeteksinya dengan sempurna.

## Verification Plan

### Automated Tests
1. Menjalankan `bun db:generate` untuk memastikan tidak ada kesalahan sintaks dan Drizzle berhasil membuat file SQL migrasinya.
2. Menjalankan `bun db:push` untuk mendorong perubahan skema baru ke PostgreSQL.
